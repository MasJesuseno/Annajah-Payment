Bab 1 — Pendahuluan

Mengenal Sistem Administrasi Sekolah SMA Annajah

1.1 Tentang Aplikasi

Sistem Administrasi Sekolah SMA Annajah adalah aplikasi berbasis web yang dirancang khusus untuk mengelola seluruh proses pembayaran di lingkungan SMA Annajah. Aplikasi ini menangani mulai dari pencatatan data siswa, pengaturan jenis pembayaran, transaksi harian, hingga pembuatan laporan keuangan.

1.2 Fitur Unggulan

🔐 Autentikasi Multi-User

Login aman dengan dua level akses: Admin dan Bendahara.

📊 Dashboard Real-time

Lihat ringkasan data pembayaran terkini dalam satu halaman.

💳 Manajemen Pembayaran

Atur berbagai jenis pembayaran: SPP, gedung, ujian, buku, dan lainnya.

📄 Kwitansi PDF Otomatis

Cetak kwitansi format A5 landscape untuk setiap transaksi.

📈 Laporan & Grafik

Rekap keuangan, grafik bulanan, distribusi pembayaran (Pie Chart & Bar Chart).

📊 Export Excel

Download laporan ke Excel dengan format rapi dan Grand Total.

Bab 2 — Instalasi & Konfigurasi

Cara install dan menjalankan aplikasi

2.1 Persiapan

Pastikan Node.js sudah terinstall di komputer. Cek dengan membuka Command Prompt (CMD) dan ketik:

node --version → Harus menampilkan versi 18+
npm --version → Harus menampilkan versi 9+

2.2 Instalasi Dependencies

Buka folder project dan jalankan perintah berikut:

1

Buka folder project
cd "D:\Aplikasi\Annajah\sma-annajah-payment\backend"

2

Install backend dependencies
npm install

3

Jalankan backend server
npm start
Server akan berjalan di http://localhost:5000

4

Buka CMD baru, install frontend
cd "D:\Aplikasi\Annajah\sma-annajah-payment\frontend"
npm install

5

Jalankan frontend
npm run dev
Aplikasi bisa diakses di http://localhost:3000

⚠️ Penting: Backend harus berjalan sebelum frontend. Jangan tutup jendela CMD backend saat menggunakan aplikasi.

2.3 Cara Alternatif (Double Click)

Buka File Explorer ke folder D:\Aplikasi\Annajah\sma-annajah-payment\ lalu double-click file install-and-run.bat. Script akan otomatis:

  1. Install semua dependencies backend & frontend
  2. Menjalankan backend di http://localhost:5000
  3. Menjalankan frontend di http://localhost:3000

2.4 Build untuk Production

Jika ingin langsung mengakses dari http://localhost:5000 (tanpa perlu frontend terpisah):

1

Build frontend
cd frontend && npm run build

2

Jalankan backend
cd backend && npm start
Akses di http://localhost:5000

💡 Database: File database sma_annajah.db akan otomatis dibuat di folder backend/ saat pertama kali server dijalankan. Data sampel (kelas, siswa, jenis pembayaran) akan otomatis terisi.

Bab 3 — Login & Autentikasi

Masuk ke aplikasi dan mengenal akun pengguna

3.1 Halaman Login

Saat pertama kali membuka aplikasi, Anda akan diarahkan ke halaman login. Tampilan login memiliki:

  • Logo SMA Annajah dengan background gradasi hijau
  • Form input Username dan Password
  • Tombol "Masuk" untuk login
  • Informasi akun demo di bagian bawah form

3.2 Akun Demo

RoleUsernamePasswordHak Akses
Admin admin admin123 Akses penuh ke semua fitur
Bendahara bendahara bendahara123 Transaksi, laporan, data siswa & kelas

3.3 Cara Login

1

Buka browser dan akses http://localhost:3000 (atau http://localhost:5000 jika sudah di-build)

2

Masukkan Username (contoh: admin) dan Password (contoh: admin123)

3

Klik tombol "Masuk"

4

Setelah berhasil, Anda akan diarahkan ke halaman Dashboard

⚠️ Jika login gagal: Pastikan username dan password benar. Password bersifat case-sensitive (huruf besar/kecil berpengaruh).

3.4 Logout

Untuk keluar dari aplikasi, klik tombol "Keluar" yang berada di bagian bawah sidebar (menu samping kiri). Anda akan dikembalikan ke halaman login.

3.5 Sesi Login

Sesi login akan berlangsung selama token JWT masih valid. Jika token kedaluwarsa atau terjadi error autentikasi (401/403), Anda akan otomatis dialihkan ke halaman login. Data login disimpan di penyimpanan lokal browser (localStorage).

Bab 5 — Manajemen Kelas

Mengelola data kelas siswa

5.1 Tentang Halaman Kelas

Halaman Kelas digunakan untuk mengelola data kelas. Kelas dikelompokkan berdasarkan tingkat: Kelas X (tingkat 10), Kelas XI (tingkat 11), dan Kelas XII (tingkat 12).

💡 Data Awal: Saat pertama kali aplikasi dijalankan, 9 kelas sampel sudah otomatis terisi (X-A, X-B, X-C, XI-A, XI-B, XI-C, XII-A, XII-B, XII-C).

5.2 Menambah Kelas Baru

1

Klik tombol "Tambah Kelas" di pojok kanan atas

2

Isi Nama Kelas (contoh: X-A, XI-B, XII-C)

3

Pilih Tingkat: Kelas X (10), Kelas XI (11), atau Kelas XII (12)

4

Klik "Tambah" untuk menyimpan

5.3 Mengedit Kelas

Arahkan mouse ke kelas yang ingin diedit, lalu klik ikon ✏️ (Edit) yang muncul. Ubah data yang diperlukan dan klik "Simpan".

5.4 Menghapus Kelas

Arahkan mouse ke kelas yang ingin dihapus, lalu klik ikon 🗑️ (Hapus).

🚫 Tidak bisa hapus: Kelas yang masih memiliki siswa tidak dapat dihapus. Pindahkan siswa ke kelas lain terlebih dahulu, atau hapus siswa dari kelas tersebut.

Bab 6 — Manajemen Siswa

Mengelola data siswa

6.1 Tentang Halaman Siswa

Halaman Data Siswa digunakan untuk mengelola seluruh data siswa SMA Annajah. Data siswa meliputi NIS, NISN, nama, jenis kelamin, kelas, tempat/tanggal lahir, alamat, dan nomor telepon.

6.2 Menambah Siswa Baru

1

Klik tombol "Tambah Siswa" di pojok kanan atas

2

Isi data siswa pada form yang muncul. Field bertanda * wajib diisi (NIS dan Nama)

3

Pilih Kelas untuk menempatkan siswa

4

Klik "Tambah Siswa" untuk menyimpan

6.3 Mencari dan Menyaring Data

Halaman Siswa dilengkapi fitur pencarian dan filter:

  • 🔍 Pencarian: Ketik nama atau NIS siswa di kolom pencarian. Data akan otomatis terfilter secara langsung.
  • 🏷️ Filter Kelas: Pilih kelas tertentu dari dropdown untuk menampilkan siswa dari kelas tersebut saja.
  • 🔄 Reset: Klik tombol "Reset" untuk menghapus semua filter dan pencarian.

6.4 Status Siswa

Bab 8 — Transaksi Pembayaran

Mencatat pembayaran, mencetak kwitansi, dan cek status SPP

8.1 Tentang Halaman Transaksi

Halaman Transaksi Pembayaran adalah halaman utama untuk mencatat pembayaran yang dilakukan oleh siswa. Setiap transaksi akan menghasilkan nomor kwitansi unik secara otomatis.

8.2 Mencatat Pembayaran Baru

1

Klik tombol "Transaksi Baru"

2

Pilih Siswa — Pilih siswa dari dropdown. Setelah memilih siswa, status SPP akan otomatis muncul jika ada.

3

Cek Status SPP — Panel hijau akan menampilkan bulan-bulan yang sudah lunas (✅) dan yang belum (❌) untuk pembayaran SPP.

4

Pilih Jenis Pembayaran — Nominal akan otomatis terisi sesuai jenis yang dipilih

5

Isi Jumlah Bayar (otomatis terisi, bisa diubah jika diperlukan)

6

Pilih Bulan Pembayaran (khusus untuk SPP)

7

Klik "Catat Pembayaran" untuk menyimpan transaksi

8.3 Nomor Kwitansi Otomatis

Setiap transaksi akan mendapatkan nomor kwitansi unik dengan format: KWI/YYYYMMDD/XXXX

  • KWI — Prefix kwitansi
  • YYYYMMDD — Tanggal transaksi
  • XXXX — Nomor urut (reset setiap hari)

Contoh: KWI/20250115/0001 (kwitansi pertama pada 15 Januari 2025)

8.4 Mencari Transaksi

Gunakan kolom pencarian di bagian atas tabel untuk mencari transaksi berdasarkan nama siswa atau NIS. Data akan terfilter secara otomatis saat Anda mengetik.

8.5 Download Kwitansi (PDF)

1

Cari transaksi yang ingin di-download kwitansinya

2

Klik ikon ⬇️ Download (warna hijau) pada baris transaksi tersebut

3

File PDF akan otomatis ter-download. Kwitansi siap dicetak!

8.6 Export Transaksi ke Excel

Di halaman Transaksi, Anda juga bisa mengexport seluruh data transaksi ke Excel. Klik tombol "Download Excel" (📊) di pojok kanan atas halaman.

File Excel akan berisi semua kolom: nomor kwitansi, tanggal, NIS, nama siswa, jenis pembayaran, bulan, jumlah, dan petugas. Bagian bawah akan menampilkan Grand Total dari seluruh transaksi.

Fitur ini berguna untuk:

  • 📋 Arsip data transaksi di spreadsheet
  • 📊 Analisis lebih lanjut dengan pivot table
  • 📎 Lampiran laporan keuangan

8.7 Menghapus Transaksi

Klik ikon 🗑️ (warna merah) pada baris transaksi yang ingin dihapus. Konfirmasi akan muncul sebelum penghapusan dilakukan.

8.7 Format Kwitansi

Kwitansi dicetak dalam format A5 Landscape dengan informasi:

  • Kop sekolah (nama SMA Annajah)
  • Nomor kwitansi dan tanggal
  • NIS dan nama siswa
  • Jenis pembayaran, periode, dan bulan
  • Jumlah pembayaran (dalam Rupiah)
  • Terbilang (nominal dalam kata-kata)
  • Tanda tangan penerima dan kepala sekolah

Bab 9 — Laporan & Export

Rekap keuangan, grafik, export Excel & cetak

9.1 Tentang Halaman Laporan

Halaman Laporan Keuangan menyediakan tiga tab laporan yang bisa dipilih: Rekap Pembayaran, Per Bulan, dan Per Siswa.

9.2 Tab: Rekap Pembayaran

Menampilkan rekap pembayaran berdasarkan jenis pembayaran, dilengkapi:

  • Filter Tanggal: Pilih rentang tanggal untuk melihat data periode tertentu
  • Pie Chart: Grafik lingkaran distribusi pembayaran per jenis
  • Tabel Detail: Nama pembayaran, jumlah transaksi, total nominal
  • Grand Total: Jumlah seluruh pembayaran di periode tersebut

9.3 Tab: Per Bulan

Menampilkan laporan pembayaran per bulan dalam satu tahun ajaran:

  • Pilih Tahun: Pilih tahun yang ingin dilihat (2024, 2025, 2026)
  • Bar Chart: Grafik batang pembayaran per bulan
  • Tabel: 12 bulan (Jan-Des) dengan jumlah transaksi dan total nominal

9.4 Tab: Per Siswa

Menampilkan riwayat pembayaran siswa tertentu:

  • Pilih Siswa: Pilih siswa dari dropdown
  • Riwayat Transaksi: Tabel tanggal, jenis pembayaran, bulan, jumlah
  • Total: Akumulasi seluruh pembayaran siswa tersebut

9.5 Cetak Laporan

Klik tombol "Cetak" (🖨️) di pojok kanan atas halaman Laporan. Browser akan menampilkan dialog cetak. Anda bisa mencetak ke printer atau menyimpan sebagai PDF.

💡 Tips Cetak: Gunakan fitur cetak browser (Ctrl+P) untuk hasil terbaik. Atur orientasi menjadi Landscape jika tabel terlalu lebar.

Bab 10 — Pengaturan Akun & Tampilan

Mengelola akun dan navigasi aplikasi

10.1 Mengubah Password

Saat ini, fitur ubah password tersedia melalui API:

1

Klik menu Dashboard untuk memastikan Anda sudah login

2

Untuk mengubah password, gunakan endpoint API:
PUT /api/auth/ubah-password
Body: { "password_lama": "admin123", "password_baru": "passwordbaru" }

⚠️ Catatan: Fitur ubah password dari antarmuka pengguna (frontend) dapat ditambahkan jika diperlukan. Saat ini tersedia melalui API.

10.2 Informasi Pengguna

Sidebar menampilkan informasi pengguna yang sedang login, meliputi:

  • Nama lengkap pengguna
  • Role (Admin / Bendahara)

10.2 Tombol Collapse Sidebar

Sidebar dapat dikecilkan (collapse) untuk memberikan lebih banyak ruang ke konten utama. Klik tombol kecil di antara sidebar dan konten (ikon < atau ☰) untuk toggle tampilan sidebar.

Saat sidebar dikecilkan, hanya ikon menu yang terlihat. Arahkan mouse (hover) untuk melihat tooltip nama menu.

10.3 Fitur yang Akan Datang

Berikut fitur yang sedang dalam pengembangan dan akan tersedia di versi mendatang:

  • 🔄 Ubah Password dari UI — Form ubah password akan ditambahkan di menu sidebar
  • ⚙️ Pengaturan Sekolah — Mengatur nama sekolah, alamat, kepala sekolah, tahun ajaran aktif
  • 📊 Grafik Dashboard Lebih Interaktif — Visualisasi data yang lebih kaya

Bab 11 — Pemecahan Masalah

Solusi untuk masalah umum

11.1 Aplikasi Tidak Bisa Diakses

MasalahPenyebabSolusi
"Connection refused"Backend belum dijalankan Jalankan npm start di folder backend
Halaman putih / blank Frontend belum di-build atau Vite belum jalan Jalankan npm run dev di folder frontend
Halaman tidak ditemukan (404) URL salah atau frontend belum di-setup Pastikan akses via http://localhost:3000 (dev) atau http://localhost:5000 (production)

11.2 Masalah Login

MasalahSolusi
"Username atau password salah" Gunakan akun demo: admin / admin123 atau bendahara / bendahara123
Terus dialihkan ke login Hapus cache browser atau hapus localStorage (F12 → Application → Local Storage → Clear)

11.3 Masalah Database

MasalahSolusi
Data hilang setelah restart File database sma_annajah.db tersimpan di folder backend/. Jangan hapus file ini! Backup secara berkala.
Error "SQLITE_BUSY" Tutup aplikasi lain yang mengakses file database yang sama. Restart backend server.
Reset database Hentikan backend, hapus file backend/sma_annajah.db, lalu jalankan backend kembali. Database akan dibuat ulang dengan data sampel.

11.4 NPM Tidak Dikenali

Jika muncul error 'npm' is not recognized, berarti Node.js belum terinstall atau tidak terdaftar di PATH.

  • Download dan install Node.js dari https://nodejs.org (versi LTS)
  • Restart komputer setelah installasi
  • Cek dengan: node --version dan npm --version

11.5 Port Sudah Dipakai

Jika port 5000 sudah dipakai aplikasi lain, ubah port di file backend/server.js:

const PORT = process.env.PORT || 5000;
Ubah menjadi: const PORT = process.env.PORT || 5001;

11.6 Backup Database

Untuk membackup data, cukup copy file backend/sma_annajah.db ke lokasi aman. Untuk merestore, copy file backup kembali ke folder backend/.

📌 Penting: Selalu backup database secara berkala! File database adalah satu-satunya tempat penyimpanan data pembayaran.

11.7 Kontak Dukungan

Jika mengalami masalah yang tidak tercantum di atas, hubungi:

  • Pengembang: Tim IT SMA Annajah
  • Dokumentasi Online: README.md di folder project
